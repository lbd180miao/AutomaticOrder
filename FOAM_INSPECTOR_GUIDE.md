# 交互式泡棉检测功能使用指南

## 功能概述

新增的交互式泡棉检测页面提供了更直观的相机操作和检测流程：

### 核心功能
1. **实时相机预览** - 查看相机实时画面
2. **点击拍照检测** - 一键拍照并进行泡棉检测
3. **导入图片检测** - 上传本地图片进行离线检测

## 访问入口

### 方式 1: 从视觉任务列表
1. 访问 `http://127.0.0.1:8000/vision/tasks/`
2. 点击顶部工具栏的 **"交互式泡棉检测"** 按钮（蓝色按钮）

### 方式 2: 直接访问
直接访问 `http://127.0.0.1:8000/vision/foam-inspector/`

## 使用流程

### 流程 1: 实时拍照检测

1. **刷新预览**
   - 点击左侧 **"刷新预览"** 按钮
   - 系统调用相机获取实时画面
   - 预览图显示在左侧区域

2. **拍照并检测**
   - 确认预览画面正确
   - 点击 **"拍照并检测"** 按钮
   - 系统自动:
     - 触发相机拍照
     - 执行泡棉检测算法
     - 在图像上绘制ROI和检测结果
     - 显示结果图和数据

3. **查看结果**
   - 右侧显示带ROI标注的结果图
   - 下方表格显示详细检测数据
   - 绿色徽章表示合格，红色表示不合格

### 流程 2: 导入图片检测

1. **选择图片**
   - 点击右上角 **"导入图片检测"** 按钮
   - 在文件选择对话框中选择图片
   - 支持格式: JPG, PNG, BMP, TIFF

2. **自动检测**
   - 图片上传后自动进行检测
   - 无需相机连接即可使用

3. **查看结果**
   - 结果显示方式与拍照检测相同

## 检测数据说明

### 数据表格字段

| 字段 | 说明 | 值 |
|------|------|-----|
| **位置索引** | 泡棉贴附位置编号 | 整数 (0, 1, 2, ...) |
| **综合分数** | 检测置信度分数 | 0.0 ~ 1.0 |
| **泡棉存在** | 是否检测到泡棉 | 是/否 |
| **位置对齐** | 泡棉是否对齐ROI | 是/否 |
| **边缘起翘** | 是否检测到起翘 | 是/否 |
| **判定结果** | 最终合格判定 | 合格/不合格 |
| **X偏移** | 水平偏移量 | 像素值 |
| **Y偏移** | 垂直偏移量 | 像素值 |
| **覆盖率** | 泡棉覆盖ROI比例 | 百分比 |

### 判定逻辑

泡棉检测合格需满足以下条件：
- ✓ 泡棉存在 (is_present = true)
- ✓ 综合分数 ≥ 阈值 (默认 0.8)
- ✓ X/Y偏移 ≤ 最大偏移 (默认 30px)
- ✓ 覆盖率 ≥ 阈值 (默认 0.75)
- ✓ 无边缘起翘 (has_lifted_edge = false)

### 缺陷类型

| 类型 | 英文代码 | 说明 |
|------|----------|------|
| 无缺陷 | NONE | 泡棉完全合格 |
| 位置偏移 | MISALIGNED | 泡棉中心偏离ROI中心 |
| 边缘起翘 | LIFTED_EDGE | 检测到泡棉边缘翘起 |
| 泡棉缺失 | MISSING | ROI内未检测到泡棉 |

## 图像标注说明

### 结果图中的可视化元素

- **绿色框** - ROI检测区域
- **蓝色框** - 检测到的泡棉位置
- **白色箭头** - 泡棉中心偏移向量
- **红色框** - 缺陷标记（如有）
- **文字标注** - 检测参数和分数

### 图例说明
- 🟩 泡棉合格区域
- 🟥 泡棉异常区域
- ⬜ ROI检测框

## API接口说明

如需集成到其他系统，可使用以下API：

### 1. 获取相机预览
```http
POST /vision/api/camera/preview/
Content-Type: application/json

Response:
{
  "success": true,
  "image_url": "/media/temp_previews/preview_xxx.jpg",
  "timestamp": "2026-06-17 08:22:48"
}
```

### 2. 拍照并检测
```http
POST /vision/api/foam/capture-inspect/
Content-Type: application/json

{
  "position_index": 0
}

Response:
{
  "success": true,
  "result": {
    "task_id": 123,
    "position_index": 0,
    "is_present": true,
    "is_aligned": true,
    "has_lifted_edge": false,
    "score": 0.96,
    "is_passed": true,
    "offset_x_px": 2.5,
    "offset_y_px": -1.2,
    "coverage_ratio": 0.85,
    "defect_type": "NONE",
    "result_image_url": "/media/vision/results/...",
    "original_image_url": "/media/vision/captures/..."

```

### 3. 上传图片检测
```http
POST /vision/api/foam/upload-inspect/
Content-Type: multipart/form-data

Form Data:
- image: (binary file)
- position_index: 0

Response: (同拍照检测)
```

## 配置参数

检测算法支持自定义配置（在代码中调整）：

```python
inspection_config = {
    'score_threshold': 0.8,      # 分数阈值 (0.0-1.0)
    'coverage_threshold': 0.75,  # 覆盖率阈值 (0.0-1.0)
    'max_offset_px': 30,         # 最大像素偏移
}
```

## 故障排查

### 问题 1: 相机预览失败
**错误**: "图像采集失败: 相机 0 保存图像失败，错误码: -2147483646"

**解决方案**:
1. 运行诊断工具: `python diagnose_hik_camera.py`
2. 检查输出目录权限
3. 确认SDK配置正确
4. 参考 `HIK_CAMERA_FIX_README.md`

### 问题 2: 页面显示空白
**检查**:
1. Django服务器是否正常运行
2. 浏览器控制台是否有JavaScript错误
3. URL路由是否正确配置

### 问题 3: 上传图片失败
**检查**:
1. 文件大小是否超限（默认2.5MB）
2. 文件格式是否支持
3. MEDIA_ROOT目录权限
