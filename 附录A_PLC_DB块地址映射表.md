## 说明
以下模板由甲方 PLC 工程师填写，填写完成后交乙方进行软件配置，双方各留存一份已确认副本。

**数据流向约定**：
- **PLC→软件**：软件从PLC读取数据
- **软件→PLC**：软件向PLC写入数据
- **PLC↔软件**：双向读写

## 通信协议说明

### 1. 通信方式
- **协议**：S7通信协议（西门子PLC）/ Modbus TCP（其他PLC）
- **网络**：工业以太网
- **IP地址**：
  - PLC IP: ________________（由甲方填写）
  - 软件服务器IP: ________________（由甲方填写）
- **端口号**：102（S7）/ 502（Modbus TCP）
- **扫描周期**：100ms（心跳1000ms）

### 2. 数据类型说明
- **Bool**: 布尔型，占1位（0/1）
- **Int**: 16位有符号整数（-32768 ~ 32767）
- **DInt**: 32位有符号整数
- **Real**: 32位浮点数（IEEE 754）
- **String**: 字符串，格式：前2字节长度+实际字符（UTF-8编码）

### 3. 信号交互时序说明

#### 3.1 条码读取流程
```
1. PLC: 扫码枪触发，读取条码
2. PLC: 写入条码字符串到Tag#1或Tag#7
3. PLC: 置位Tag#2（产品）或Tag#8（料框）= 1
4. 软件: 检测到Tag#2=1，读取Tag#1条码
5. 软件: 处理完成后，置位Tag#4或Tag#10 = 1
6. PLC: 检测到Tag#4=1，清除Tag#2，进入下一流程
7. 软件: 检测到Tag#2=0，清除Tag#4
```

#### 3.2 视觉定位流程
```
1. PLC: 料框定位机构完成，置位Tag#19 = 1
2. 软件: 检测到Tag#19=1，触发深度相机拍照
3. 软件: 完成拍照，置位Tag#20 = 1
4. 软件: 计算补偿值，写入Tag#21-26
5. 软件: 置位Tag#27（定位完成）、Tag#28-30（定位结果）、Tag#31（可装箱）= 1
6. PLC: 读取补偿值，应用到机器人坐标
7. PLC: 置位Tag#32 = 1（补偿已应用）
8. PLC: 清除Tag#19，进入装箱流程
9. 软件: 检测到Tag#19=0，清除Tag#20、Tag#27、Tag#31
```

#### 3.3 泡棉检测流程
```
1. PLC: 机器人完成泡棉贴附，置位Tag#40 = 1
2. 软件: 检测到Tag#40=1，触发2D相机拍照
3. 软件: 完成拍照，置位Tag#41 = 1
4. 软件: 执行检测算法，判定合格/不合格
5. 软件: 置位Tag#42（合格）或Tag#43（不合格）= 1
6. 软件: 写入检测数据到Tag#46-49
7. 软件: 置位Tag#44（检测完成）= 1
8. PLC: 读取检测结果，根据Tag#42/43决定继续或报警
9. PLC: 置位Tag#45 = 1（结果已接收）
10. PLC: 清除Tag#40
11. 软件: 检测到Tag#40=0，清除Tag#41-44
```

#### 3.4 心跳监测机制
```
- PLC和软件各自每秒翻转一次心跳位
- 对方监测心跳位在3秒内无变化，判定为通信异常
- 软件检测到PLC心跳丢失：记录报警，停止处理PLC请求
- PLC检测到软件心跳丢失：触发报警灯，停止发送新请求
```

## 特殊说明

### 1. 字符串编码
- 所有字符串使用UTF-8编码
- 字符串最大长度已在"长度"列标注
- 字符串格式：前2字节为实际长度，后续为字符数据

### 2. 浮点数精度
- 补偿值精度：0.1mm
- 覆盖率精度：0.01（1%）
- 所有浮点数使用IEEE 754标准

### 3. 信号复位规则
- 单次信号：完成后由触发方复位
- 状态信号：状态改变时更新
- 报警信号：确认后由软件复位

### 4. 扩展性
- DB108预留了扩展空间
- 如需新增信号，优先使用预留区域
- 新增信号需双方确认后更新本表
