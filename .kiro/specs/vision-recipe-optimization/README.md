# Vision Recipe Optimization - Implementation Plan

## 项目概述

优化视觉配方模块，实现泡棉检测工作台与配方管理的数据互通，提供用户友好的配方选择、标定和保存机制。

## 文档结构

- **[requirements.md](./requirements.md)** - 详细需求文档 (12个核心需求，52个验收标准)
- **[design.md](./design.md)** - 技术设计文档 (架构、API、算法、业务流程)
- **[tasks.md](./tasks.md)** - 实施任务列表 (23个任务，8个阶段)

## 核心功能

✅ **配方自动加载** - 工作台根据 POS 自动加载对应配方  
✅ **ROI 可视化** - 在相机画面上显示配方的左右泡棉 ROI 框  
✅ **灵活保存** - 标定 ROI 后可选择保存到任意 POS (0/1/2)  
✅ **临时切换** - 支持临时使用其他配方测试，不影响当前设置  
✅ **检测集成** - 检测自动使用配方参数，结果显示配方信息  
✅ **跨页面同步** - 配方管理页面和工作台数据完全共通  
✅ **向后兼容** - 无配方时仍可使用手动标定逻辑  

## 技术亮点

- **零破坏性改动** - 保持现有页面布局和 API 兼容
- **智能优先级** - 配方 > 标定配置 > 默认值
- **性能优化** - 前端缓存配方，避免重复查询
- **数据验证** - ROI 范围和阈值完整性校验

## 实施路线图

### 阶段 1-2: 基础架构 (7小时)
- 扩展后端 API 接收 recipe_id
- 实现前端配方状态管理
- 加载并缓存配方数据

### 阶段 3-4: 可视化 (5.5小时)
- 添加配方信息卡片
- 在相机画面绘制 ROI 框
- POS 切换自动更新显示

### 阶段 5-6: 交互功能 (5小时)
- 实现 POS 选择对话框
- 保存标定 ROI 到指定配方
- 临时切换配方功能

### 阶段 7: 检测集成 (3.5小时)
- 拍照和上传检测携带 recipe_id
- 结果显示配方信息
- 清除临时选择

### 阶段 8: 测试验证 (11小时)
- 单元测试和集成测试
- 端到端流程验证
- 向后兼容性回归测试

**总计**: ~35 小时 (约 4.5 个工作日)
