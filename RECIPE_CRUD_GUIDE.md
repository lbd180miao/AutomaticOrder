# 视觉配方增删改查功能 - 使用指南

## 📋 功能概述

已为视觉配方管理页面添加完整的增删改查（CRUD）功能，支持灵活管理不同层数的泡棉检测配方。

## ✨ 新增功能

### 1. **新增配方**
- 点击页面右上角的「➕ 新增配方」按钮
- 在弹窗中填写：
  - **配方名称**：例如"第5层泡棉检测配方"
  - **POS 位置**：对应料架的层数（从 0 开始，0=第1层，1=第2层，以此类推）
  - **备注**：可选，用于记录配方说明
- 点击「确定」创建配方
- 新配方将使用默认的 ROI 和阈值参数

### 2. **编辑配方**
- 在配方列表中找到要编辑的配方
- 点击「编辑」按钮
- 在下方的「ROI / 阈值微调」区域修改参数：
  - **左右泡棉 ROI**：x, y, width, height（像素坐标）
  - **阈值配置**：覆盖率、得分、X/Y偏移阈值
- 点击「保存当前配方」按钮保存修改

### 3. **复制配方**
- 在配方列表中找到要复制的配方
- 点击「复制」按钮
- 在弹窗中：
  - 配方名称会自动填充为"原名称 (副本)"
  - 输入新的 POS 位置（不能与现有配方重复）
  - 修改备注（可选）
- 点击「确定」创建副本
- 副本将继承原配方的所有 ROI 和阈值参数

### 4. **删除配方**
- 在配方列表中找到要删除的配方
- 点击红色的「删除」按钮
- 在确认对话框中确认删除
- 配方将被永久删除（无法恢复）

### 5. **查看配方**
- 配方列表自动显示所有激活的配方
- 每个配方卡片显示：
  - POS 位置和层数
  - 左右泡棉的 ROI 坐标
  - 阈值配置（覆盖率、得分）
  - 当前使用状态（蓝色边框表示正在使用）

## 🔧 技术实现

### 后端 API

#### 1. 创建配方
```http
POST /vision/api/recipes/foam-2d/create/
Content-Type: application/json

{
  "name": "第5层泡棉检测配方",
  "pos": 4,
  "remark": "用于5层料架的泡棉检测"
}
```

#### 2. 更新配方
```http
POST /vision/api/recipes/foam-2d/save/
Content-Type: application/json

{
  "id": 123,
  "pos": 4,
  "name": "第5层泡棉检测配方",
  "roi_config": {
    "leftFoamROI": {"x": 220, "y": 140, "width": 90, "height": 70},
    "rightFoamROI": {"x": 780, "y": 140, "width": 110, "height": 70}
  },
  "threshold_config": {
    "coverage_threshold": 0.75,
    "score_threshold": 0.8,
    "max_offset_px": 30
  }
}
```

#### 3. 删除配方
```http
POST /vision/api/recipes/foam-2d/{recipe_id}/delete/
```

#### 4. 获取配方列表
```http
GET /vision/api/recipes/?recipe_type=FOAM_2D&is_active=true
```

### 前端函数

- `openCreateModal()` - 打开新增配方弹窗
- `copyRecipe(recipeId)` - 复制指定配方
- `deleteRecipe(recipeId, recipeName)` - 删除指定配方
- `selectRecipeForEdit(recipeId)` - 选择配方进行编辑
- `saveSelectedRecipe()` - 保存当前编辑的配方
- `closeModal()` - 关闭弹窗
- `submitModal()` - 提交新增配方

## 🎯 使用场景

### 场景1：新增4层料架配方
1. 当前系统只有3层料架配方（POS 0, 1, 2）
2. 需要添加第4层配方（POS 3）
3. 点击「➕ 新增配方」
4. 填写名称"第4层泡棉检测配方"，POS 填 3
5. 创建后，在编辑区域调整 ROI 坐标
6. 点击「保存当前配方」

### 场景2：复制配方到新层
1. 第3层和第4层的泡棉位置相同
2. 选择第3层配方（POS 2），点击「复制」
3. 在弹窗中输入 POS 3
4. 创建副本，无需重新标定 ROI

### 场景3：删除测试配方
1. 测试阶段创建了多个配方
2. 找到测试配方，点击「删除」
3. 确认删除
4. 配方从列表中移除

### 场景4：调整现有配方
1. 发现第2层的 ROI 位置不准确
2. 点击第2层配方的「编辑」
3. 在 ROI 表单中修改坐标
4. 点击「保存当前配方」
5. 配方立即生效

## ⚠️ 注意事项

1. **POS 唯一性**
   - 每个 POS 位置只能有一个激活的配方
   - 新增配方时，如果 POS 已存在，会提示错误
   - 可以先删除旧配方，再创建新配方

2. **配方删除**
   - 删除操作不可撤销
   - 删除后相关的检测历史记录不受影响
   - 建议删除前确认该配方不再使用

3. **默认配方**
   - 点击「补齐默认配方」会为 POS 0-2 创建默认配方
   - 如果某个 POS 已有配方，不会重复创建
   - 默认配方使用标准的 ROI 和阈值参数

4. **配方生效**
   - 保存配方后立即生效
   - 泡棉检测工作台会自动加载对应 POS 的配方
   - 无需重启服务
